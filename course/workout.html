Bootstrap process

The question is what is really happening when angular.js is loaded on our page. Let's follow the happenings from the point when DOMContentLoaded event is fired in the browser, Angular has a listener to that event which kicks the whole framework in motion. First of all Angular traverse through the DOM to find the ng-app directive, that element which contain will be the root element of our app. Then Angular loads the module definition referenced in ng-app and creates the injector which will be used for the DI. The injector creates the $rootScope object which will be the default context of our application. And the last step is that Angular goes through the DOM tree starting from the root element, setting up the bindings and compiles directives. And then just waits for the first event to come. This is the big picture, that you need to see first. Question is what happens if you have no ng-app in the template.
